Basic Animation Guide

Get started with Dioxus Motion animations through simple examples. This guide covers the fundamentals you need to create your first animations.

Quick Reference

1. Create a motion value

let mut value = use_motion(0.0f32);

2. Animate the value

value.animate_to(100.0, config);

Step 1: Your First Animation

Let's create a simple animation that moves a progress bar from 0% to 100%.

The Code

// 1. Import the prelude
use dioxus_motion::prelude::*;

// 2. Create a motion value
let mut value = use_motion(0.0f32);

// 3. Animate the value
value.animate_to(
    100.0,
    AnimationConfig::new(AnimationMode::Tween(Tween {
        duration: Duration::from_millis(1000),
        easing: easer::functions::Linear::ease_in_out,
    })),
);

What's happening here?

  • We create a motion value starting at 0
  • We animate it to 100 over 1 second
  • We use a linear easing function for smooth motion

  • Different value types: You can animate f32, f64, Transform, Color, and custom types.
  • Reading values: Use value.get_value() to access the current value at any time.

Step 2: Tween vs Spring Animations

Compare the two main animation types: Tween (time-based) and Spring (physics-based).

Tween Animation

  • Time-based with fixed duration
  • Precise control over timing
  • Uses easing functions
  • Best for UI transitions

Spring Animation

  • Physics-based, no fixed duration
  • Natural, organic motion
  • Configurable stiffness & damping
  • Best for interactive elements

Tween Example

AnimationMode::Tween(Tween { duration: Duration::from_millis(1000), easing: easer::functions::Cubic::ease_in_out })

Spring Example

AnimationMode::Spring(Spring { stiffness: 100.0, // Spring force damping: 10.0, // Bounce reduction mass: 1.0 // Weight })

When to use which?

Use Tween for:

  • Page transitions
  • Fade effects
  • Precise timing requirements

Use Spring for:

  • Draggable elements
  • Interactive UI
  • Natural-feeling animations

Interactive Animation Guide

Learn how to create beautiful animations in Dioxus Motion through interactive examples. Start with basic animations and progress to more complex ones.

Core Animation Types

  • Tween: Time-based animations with precise duration and easing
  • Spring: Physics-based animations that feel natural and responsive

Advanced Features

  • Transform: Built-in type for position, scale, and rotation
  • Custom Types: Create your own animatable types
  • Sequences: Chain multiple animations for complex, coordinated motion

1. Basic Tween Animation

Time-based animations with precise control over duration and easing. Perfect for fade effects and smooth transitions.

// Initialize the motion value
let mut opacity = use_motion(0.0f32);

// Option 1: Trigger on mount
use_effect(move || {
    opacity.animate_to(
        1.0,
        AnimationConfig::new(AnimationMode::Tween(Tween {
            duration: Duration::from_millis(500),
            easing: easer::functions::Cubic::ease_in_out,
        })),
    );
});

// Option 2: Trigger on state change
let mut is_visible = use_signal(|| false);
use_effect(move || {
    if *is_visible.read() {
        opacity.animate_to(1.0, /* config */);
    } else {
        opacity.animate_to(0.0, /* config */);
    }
});

// Option 3: Trigger on event
rsx! {
    button {
        onclick: move |_| {
            opacity.animate_to(1.0, /* config */);
        },
        "Animate"
    }
}

2. Spring Animation

Physics-based animations that create natural motion. Great for interactive elements that need organic movement.


#[component]
fn TransformAnimation() -> Element {
    let mut transform = use_motion(Transform::new(0.0, 0.0, 1.0, 0.0));
    let mut is_animated = use_signal(|| false);

    use_effect(move || {
        if *is_animated.read() {
            transform.animate_to(
                Transform::new(100.0, 50.0, 1.2, 45.0),
                AnimationConfig::new(AnimationMode::Spring(Spring {
                    stiffness: 100.0,
                    damping: 10.0,
                    mass: 1.0,
                    velocity: 0.0,
                })),
            );
        } else {
            transform.animate_to(
                Transform::new(0.0, 0.0, 1.0, 0.0),
                AnimationConfig::new(AnimationMode::Spring(Spring {
                    stiffness: 100.0,
                    damping: 10.0,
                    mass: 1.0,
                    velocity: 0.0,
                })),
            );
        }
    });

    let transform_style = use_memo(move || {
        format!(
            "transform: translate({}px, {}px) scale({}) rotate({}deg);",
            transform.get_value().x,
            transform.get_value().y,
            transform.get_value().scale,
            transform.get_value().rotation * 180.0 / std::f32::consts::PI
        )
    });

    rsx! {
        div { class: "space-y-4 w-full text-center",
            button {
                class: "px-4 py-2 bg-primary/20 hover:bg-primary/30 rounded-lg text-primary transition-colors",
                onclick: move |_| is_animated.toggle(),
                if *is_animated.read() { "Reset" } else { "Animate" }
            }
            div {
                class: "w-32 h-32 mx-auto bg-primary rounded-lg",
                style: "{transform_style}"
            }
        }
    }
}

3. Transform Animation

Built-in Transform type for animating position, scale, and rotation. Uses the same animation modes as basic values.

// Transform combines multiple properties:
// - x, y: Position
// - scale: Size
// - rotation: Angle in radians
let mut transform = use_motion(Transform::new(0.0, 0.0, 1.0, 0.0));

// Animate with spring for natural motion
transform.animate_to(
    Transform::new(100.0, 50.0, 1.2, 45.0),
    AnimationConfig::new(AnimationMode::Spring(Spring {
        stiffness: 100.0,
        damping: 10.0,
        mass: 1.0,
        velocity: 0.0,
    })),
);

// Or use tween for precise timing
transform.animate_to(
    Transform::new(0.0, 0.0, 1.0, 0.0),
    AnimationConfig::new(AnimationMode::Tween(Tween {
        duration: Duration::from_millis(300),
        easing: easer::functions::Cubic::ease_out,
    })),
);

4. Custom Animation Type

Create your own animatable types by implementing the Animatable trait. This example shows color interpolation.

#[derive(Debug, Copy, Clone)]
struct ColorValue {
    r: f32, g: f32, b: f32,
}

// Implement Animatable to enable animation
impl Animatable for ColorValue {
    fn zero() -> Self { ColorValue { r: 0.0, g: 0.0, b: 0.0 } }
    fn epsilon() -> f32 { 0.001 }
    fn magnitude(&self) -> f32 {
        (self.r * self.r + self.g * self.g + self.b * self.b).sqrt()
    }
    fn interpolate(&self, target: &Self, t: f32) -> Self {
        ColorValue {
            r: self.r + (target.r - self.r) * t,
            g: self.g + (target.g - self.g) * t,
            b: self.b + (target.b - self.b) * t,
        }
    }
}

// Use it like any other motion value
let mut color = use_motion(ColorValue { r: 0.2, g: 0.5, b: 0.8 });
color.animate_to(
    ColorValue { r: 0.8, g: 0.3, b: 0.2 },
    AnimationConfig::new(AnimationMode::Spring(Spring::default())),
);

5. Animation Sequences

Chain multiple animations together to create complex, coordinated motion. Perfect for multi-step animations and interactive counters.

// Initialize multiple motion values
let mut value = use_motion(0.0f32);
let mut scale = use_motion(1.0f32);
let mut count = use_signal(|| 0);

// Create and trigger a sequence on button click
let onclick = move |_| {
    // Create a new sequence that animates based on count
    let sequence = AnimationSequence::new()
        .then(
            (count + 1) as f32 * 100.0,  // Dynamic target value
            AnimationConfig::new(AnimationMode::Spring(Spring {
                stiffness: 180.0,
                damping: 12.0,
                mass: 1.0,
                velocity: 10.0,
            }))
        );

    // Animate scale independently
    scale.animate_to(
        1.2,
        AnimationConfig::new(AnimationMode::Spring(Spring::default()))
    );

    // Start the sequence animation
    value.animate_sequence(sequence);
    count += 1;
}

// Use the animated values in your component
rsx! {
    div {
        style: "transform: translateY({value.get_value()}px) scale({scale.get_value()})",
        "Count: {count}"
    }
    button {
        onclick: onclick,
        "Increment"
    }
}

5. SIMD Batch Color Animation

Demonstrates SIMD-accelerated batch color animation using MotionBatch<Color>. Animates 4 colors in parallel for high-performance, parallelized effects.

// Animate 4 colors in parallel using SIMD
let mut batch = MotionBatch::new([
    Color::new(1.0, 0.0, 0.0, 1.0), // Red
    Color::new(0.0, 1.0, 0.0, 1.0), // Green
    Color::new(0.0, 0.0, 1.0, 1.0), // Blue
    Color::new(1.0, 1.0, 0.0, 1.0), // Yellow
], AnimationConfig::new(AnimationMode::Spring(Spring::default())));

// Animate to new colors in parallel
batch.animate_to([
    Color::new(0.0, 1.0, 1.0, 1.0), // Cyan
    Color::new(1.0, 0.0, 1.0, 1.0), // Magenta
    Color::new(1.0, 1.0, 1.0, 1.0), // White
    Color::new(0.0, 0.0, 0.0, 1.0), // Black
], AnimationConfig::new(AnimationMode::Spring(Spring::default())));

6. Advanced Animation Features

Explore additional features like loops, delays, and completion callbacks for more control over your animations.

// Loop animations infinitely or a specific number of times
value.animate_to(
    1.0,
    AnimationConfig::new(AnimationMode::Tween(Tween::default()))
        .with_loop(LoopMode::Infinite)  // Loop forever
);

value.animate_to(
    1.0,
    AnimationConfig::new(AnimationMode::Tween(Tween::default()))
        .with_loop(LoopMode::Times(3))  // Loop 3 times
);

// Add delays before starting animations
value.animate_to(
    1.0,
    AnimationConfig::new(AnimationMode::Spring(Spring::default()))
        .with_delay(Duration::from_secs(1))  // Wait 1 second before starting
);

// Execute callbacks when animations complete
value.animate_to(
    1.0,
    AnimationConfig::new(AnimationMode::Spring(Spring::default()))
        .with_on_complete(|| println!("Animation complete!"))
);

Best Practices

Performance

  • Use spring animations for natural motion
  • Keep tween durations under 300ms for snappy feedback
  • Avoid animating too many elements simultaneously
  • Use transform instead of position for better performance

UX Guidelines

  • Maintain consistent animation durations
  • Use easing functions that match your app's personality
  • Provide visual feedback for user interactions
  • Consider reduced motion preferences
